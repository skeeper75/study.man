# 정렬과 페이징

`ORDER BY`로 결과를 정렬하고, `LIMIT`과 `OFFSET`으로 원하는 범위의 데이터만 가져올 수 있습니다.

## ORDER BY - 정렬

```sql
-- 오름차순 (기본값)
SELECT * FROM products ORDER BY price ASC;

-- 내림차순
SELECT * FROM products ORDER BY price DESC;

-- 여러 컬럼으로 정렬
SELECT * FROM products
ORDER BY category ASC, price DESC;
```

## NULLS FIRST / NULLS LAST

NULL 값의 정렬 위치를 지정할 수 있습니다:

```sql
-- NULL을 마지막에 배치 (기본값: ASC에서는 LAST)
SELECT * FROM products ORDER BY stock NULLS LAST;

-- NULL을 처음에 배치
SELECT * FROM products ORDER BY stock DESC NULLS FIRST;
```

## LIMIT - 결과 개수 제한

```sql
-- 상위 3개만 조회
SELECT * FROM products
ORDER BY price DESC
LIMIT 3;
```

## OFFSET - 건너뛰기

```sql
-- 2개를 건너뛰고 3개 조회
SELECT * FROM products
ORDER BY price DESC
LIMIT 3 OFFSET 2;
```

## 페이지네이션 패턴

```sql
-- 페이지 크기: 10, 페이지 번호: 3
SELECT * FROM products
ORDER BY id
LIMIT 10 OFFSET 20;  -- (page - 1) * page_size
```

<Callout type="warning">
OFFSET 기반 페이지네이션은 대용량 데이터에서 느려집니다. OFFSET이 클수록 건너뛰어야 할 행이 많아지기 때문입니다. 대안으로 커서 기반 페이지네이션을 사용하세요. [실전: 페이지네이션](/practical/pagination)에서 자세히 다룹니다.
</Callout>

## FETCH FIRST (SQL 표준)

`LIMIT`은 PostgreSQL/MySQL에서 지원하는 비표준 문법입니다. SQL 표준 문법은 다음과 같습니다:

```sql
SELECT * FROM products
ORDER BY price DESC
FETCH FIRST 5 ROWS ONLY;

-- OFFSET과 함께
SELECT * FROM products
ORDER BY price DESC
OFFSET 10 ROWS
FETCH FIRST 5 ROWS ONLY;
```
