# PostgreSQL 스터디 사이트 - 제품 개요

## 프로젝트 개요

PostgreSQL 스터디 사이트는 PostgreSQL 데이터베이스를 처음 접하는 개발자부터 실무에서 활용하는 중급 개발자까지 체계적으로 학습할 수 있도록 설계된 교육용 웹 플랫폼입니다.

단순한 문서 나열이 아니라, 사용자가 실제 문제를 해결하는 과정에서 자연스럽게 PostgreSQL을 익힐 수 있는 경험을 제공합니다. 당근마켓과 토스의 제품 철학에서 영감을 받아, 불필요한 복잡성을 제거하고 학습 본질에 집중하는 인터페이스를 지향합니다.

---

## 목표

### 핵심 목표

- **접근성**: SQL 초보자가 15분 안에 첫 쿼리를 실행해볼 수 있는 환경 제공
- **깊이**: 기초부터 성능 최적화, 고급 기능까지 단계별 커리큘럼 구성
- **실용성**: 실제 서비스에서 쓰이는 패턴과 예제 중심의 콘텐츠
- **신뢰성**: 공식 문서 기반의 정확한 정보와 검증된 예제 제공

### 성공 지표

- 신규 방문자의 첫 번째 학습 완료율 > 60%
- 월간 재방문율 > 40%
- 검색을 통한 원하는 내용 탐색 성공률 > 85%

## 타겟 사용자

### 주요 타겟

| 페르소나 | 설명 | 필요사항 |
|---|---|---|
| SQL 입문자 | 데이터베이스를 처음 배우는 개발 학습자 | 친절한 설명, 시각적 예제, 단계별 가이드 |
| 주니어 개발자 | 실무에서 PostgreSQL을 사용하기 시작한 1-3년차 | 실전 패턴, 트러블슈팅, 성능 팁 |
| 백엔드 개발자 | ORM 너머의 SQL을 직접 다루고 싶은 개발자 | 고급 기능, 최적화, 아키텍처 패턴 |
| 데이터 분석가 | 분석 쿼리와 집계 함수를 학습하려는 사용자 | 분석 함수, CTE, 윈도우 함수 |

### 사용자 여정

```
발견 → 온보딩 → 첫 학습 → 반복 학습 → 심화 학습 → 커뮤니티 참여
```

## 핵심 기능

### 1. 단계별 학습 커리큘럼

- **기초 과정**: SQL 기본 문법, 데이터 타입, CRUD 연산
- **중급 과정**: 조인, 서브쿼리, 인덱스, 트랜잭션
- **고급 과정**: 성능 최적화, 파티셔닝, JSON 처리, 전문 검색
- **실전 과정**: 실제 서비스 시나리오 기반 문제 해결

### 2. 인터랙티브 실습 환경

- 브라우저에서 직접 SQL을 실행해볼 수 있는 샌드박스
- 사전 로드된 샘플 데이터셋 (전자상거래, SNS, 로그 데이터 등)
- 즉각적인 실행 결과와 실행 계획(EXPLAIN) 시각화

### 3. 스마트 검색

- 개념, 함수명, 에러 메시지로 빠른 탐색
- 연관 주제 자동 추천
- 최근 학습 내용 기반 개인화 추천

### 4. 레퍼런스 문서

- PostgreSQL 공식 문서와 연계된 상세 레퍼런스
- 버전별 기능 비교 (PostgreSQL 13 ~ 17)
- 자주 발생하는 에러와 해결책 모음

### 5. 학습 진도 관리 (향후 확장)

- 학습 완료 현황 시각화
- 북마크 및 노트 기능
- 학습 뱃지 및 성취 시스템

## 설계 철학

### 당근마켓에서 가져온 원칙: 따뜻하고 친근한 전문성

당근마켓은 복잡한 중고거래를 동네 이웃과의 대화처럼 단순하게 만들었습니다. 이 사이트도 어렵게 느껴지는 데이터베이스 개념을 "이웃에게 설명하듯" 풀어냅니다.

- **친근한 언어**: 기술 용어를 쉬운 비유와 함께 설명
- **점진적 복잡성**: 처음엔 단순하게, 필요할 때 깊이 들어가는 구조
- **빠른 성공 경험**: 처음 시작하는 사람이 5분 안에 결과물을 얻도록 설계

### 토스에서 가져온 원칙: 신뢰와 명확함

토스는 금융이라는 복잡한 도메인을 명확하고 신뢰할 수 있는 경험으로 만들었습니다.

- **정확성 우선**: 잘못된 정보는 없다. 불확실한 내용은 명확히 표시
- **일관된 경험**: 어느 페이지를 방문해도 동일한 품질과 구조
- **투명한 근거**: 공식 문서 출처 명시, 버전 정보 표기
- **미니멀한 UI**: 학습 콘텐츠에 집중할 수 있도록 불필요한 요소 제거

### UI/UX 원칙

- **가독성**: 코드 블록과 텍스트의 명확한 구분, 충분한 여백
- **접근성**: WCAG 2.1 AA 수준의 색상 대비와 키보드 탐색 지원
- **반응형**: 모바일에서도 코드와 설명이 읽기 편한 레이아웃
- **다크 모드**: 개발자 친화적인 다크/라이트 테마 전환

## 콘텐츠 구조

```
학습 콘텐츠
├── 시작하기
│   ├── PostgreSQL이란?
│   ├── 설치 및 환경 설정
│   └── 첫 번째 데이터베이스 만들기
├── SQL 기초
│   ├── SELECT와 데이터 조회
│   ├── 데이터 삽입/수정/삭제
│   ├── 조건 필터링 (WHERE)
│   └── 정렬과 제한 (ORDER BY, LIMIT)
├── 데이터 모델링
│   ├── 테이블과 데이터 타입
│   ├── 제약 조건
│   └── 관계와 외래키
├── 중급 SQL
│   ├── 조인 (JOIN)
│   ├── 집계 함수와 GROUP BY
│   ├── 서브쿼리와 CTE
│   └── 윈도우 함수
├── PostgreSQL 고급 기능
│   ├── 인덱스 전략
│   ├── 트랜잭션과 잠금
│   ├── JSON/JSONB 처리
│   ├── 전문 검색 (Full-text Search)
│   └── 파티셔닝
└── 실전 패턴
    ├── 성능 최적화
    ├── 쿼리 분석 (EXPLAIN ANALYZE)
    └── 실서비스 시나리오
```
